# Тестовое задание: Автотесты для API Авито

Данный проект содержит набор автоматизированных функциональных и E2E тестов для проверки API сервиса объявлений.
Описание: Задание 1 находится в `TASK_1.md`, баги в `BUGS.md`, тест-кейсы в `TESTCASES.md`,
allure отчет в `/allure-report/index.html`, тесты в папке `tests`
Также доступна последняя версия отчета на https://vergil4828.github.io/Avito-qa/ , прогон тестов доступен в пайплайне

## Используемый стек

* **Python 3.10+**
* **Pytest**
* **Poetry** — управление зависимостями и виртуальным окружением
* **Allure** — генерация отчетов

---

## Установка окружения и зависимостей

Вы можете выбрать один из двух способов установки проекта.
Для каждого из них требуется созданное виртуальное окружение на базе python 3.10

### Способ 1: Использование Poetry (Рекомендуемый)

Если у вас установлен Poetry, выполните в корне проекта в терминале:

```bash
# Установка всех зависимостей
poetry install --no-root
```

### Способ 2: Использование pip

Используется создание venv и установка с помощью pip в терминале:

```bash
# Установка всех зависимостей
pip install -r requirements.txt
```

## Запуск тестов и линтеров

В проекте реализован многопоточный запуск. Хочу обратить внимание, что **E2E тесты** находятся в общих папках вместе с
функциональными тестами.

Для запуска тестов с генерацией Allure-отчета выполните в терминале:

```bash
pytest -n 4 --alluredir=allure-results --clean-alluredir -v
```

Если нужно сформировать отчет, то выполните в терминале:

```bash
allure serve allure-results
```

Для запуска линтера `flake8` выполните в терминале:

```bash
flake8 .
```

Для запуска форматтера `black` выполните в терминале:

```bash
black .
```

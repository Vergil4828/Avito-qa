# POST /api/1/item - Создание объявления

### Тест-кейс: Создание одиночного объявления и дубликатов объявлений

**Описание**  
Проверка успешного создания одного объявления с валидными данными, а также возможности создания дубликатов объявлений

**Предусловие**  
Сгенерировать шаблон объявления

**Шаги воспроизведения**

    1. Отправить POST-запрос на создание одиночного объявления (параметр `is_duplicate=False`)
    2. Отправить от 2 до 4 идентичных POST-запросов подряд с тем же набором данных (параметр `is_duplicate=True`)
    3. Выполнить проверку статус-кодов и структуры JSON-ответа для всех запросов

**Постусловие**  
Удалить все созданные в рамках теста объявления через ручку DELETE

**Ожидаемый результат**

    1. При `is_duplicate=False` возвращается статус-код `200 OK` и корректный JSON с ID
    2. При `is_duplicate=True` каждый из 2–4 запросов возвращает статус-код `200 OK`, система позволяет создавать дубликаты
    3. Структура всех ответов валидна, содержит сообщение о сохранении и уникальный идентификатор

---

### Тест-кейс: Создание объявления с некорректными данными (пустые/нулевые поля)

**Описание**  
Проверка обработки ошибок при попытке создания объявления с отсутствующими обязательными полями, пустыми строками или
нулевыми значениями

**Предусловие**  
Сгенерировать шаблон объявления

**Шаги воспроизведения**

    1. Выбрать одно из обязательных полей (`name`, `price`, `sellerID`, `statistics`, `likes`, `viewCount`, `contacts`)
    2. Модифицировать поле в шаблоне:
        - Установить значение `None` (отсутствующее поле, параметр `is_zero_int_or_str=False`)
        - Установить пустое значение (пустая строка для `Name` или `0` для числовых полей, `is_zero_int_or_str=True`)
    3. Отправить POST-запрос с модифицированным шаблоном
    4. Проверить статус-код ответа и текст сообщения об ошибке

**Постусловие**  
Отсутствует

**Ожидаемый результат**

    1. Сервер возвращает статус-код `400 Bad Request`
    2. В теле ответа содержится JSON с описанием ошибки валидации конкретного поля

### Тест-кейс: Создание объявления с отрицательными числовыми значениями

**Ожидаемый результат**

# GET /api/1/item/:notification_id - Получение объявления по его id

### Тест-кейс: Получение созданного объявления и проверка дубликатов по id

**Описание**  
Проверка соответствия данных в созданном объявлении при его получении по id, а также проверка уникальности id для
дубликатов объявлений

**Предусловие**

    1. Создать одиночное объявление через POST-запрос (параметр `is_duplicate=False`)
    2. Создать группу от 2 до 4 идентичных объявлений-дубликатов через POST-запросы (параметр `is_duplicate=True`)

**Шаги воспроизведения**

    1. Для каждого созданного объявления отправить GET-запрос по его уникальному id
    2. Сравнить данные в теле ответа GET-запроса с данными, которые отправлялись в POST-запросе

**Постусловие**  
Удалить все созданные в рамках теста объявления через ручку DELETE

**Ожидаемый результат**

    1. Каждый GET-запрос возвращает статус-код `200 OK`
    2. Поля в ответе полностью идентичны переданным при создании
    3. Время создания объявления высчитывается корректно
    4. Система возвращает корректные данные для каждого дубликата по его уникальному идентификатору
